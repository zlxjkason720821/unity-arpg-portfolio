## 〇、开录前 10 分钟检查清单（必须做完再按录）

### 环境

1. Unity Hub 打开工程 `unity-arpg-portfolio`，版本 **6000.5.x**
2. 打开场景：`Assets/Scenes/Demo`
3. 菜单确认存在：`ARPG Tools > Skill Editor / Level Editor / Create Sample Data`

6. Game 视图点一下获得焦点；Layout 建议：左边 Hierarchy，中间 Scene|Game，右边 Inspector，底 Project|Console

### 【A2】0:18–0:28｜代码特写证明「设计合理」（职责1 · 核心加分秒）

**目的**  
用约 10 秒代码画面，证明「技能是数据，执行器是解释器」——比多打十秒怪更有技术说服力。

**操作步骤（精确）**

1. 停止 Play（或保持 Play 但切到代码窗口；建议 Stop 更清晰）  
2. Project → `Assets/Scripts/Skills/SkillData.cs` → 双击用编辑器打开  
3. 再打开 `Assets/Scripts/Skills/SkillExecutor.cs`（分屏或左右两个标签）  
4. 在 `SkillData` 里用鼠标缓慢划过字段：`baseDamage`、`cooldown`、`range`、`radius`、`vfxPrefab`  
5. 切到 `SkillExecutor`，划过方法：`TryCast` → 强调注释里 *reads SkillData only*  
6. （可选）停在 XML 注释：`skills are data; the executor is an interpreter`

**字幕**  
`技能参数在 SkillData（ScriptableObject）`  
`SkillExecutor 只读取并执行 —— 不硬编码伤害/冷却/范围`  
`Skills are data; the executor is an interpreter.`

**旁白**  
「伤害、冷却、范围、特效引用，全部放在 ScriptableObject 资产里。运行时代码只负责读取和执行。改数值不用改代码，普攻和技能共用同一套执行器——这是面向对象里『数据与逻辑分离』的落地。」

**镜头建议**  
- 鼠标高亮要慢，停留够字幕读完  
- 不要快速滚屏

### 【A3】0:28–0:48｜AI 状态机 + Gizmos（职责1 · 可视化状态）

**录前设置**

1. Play  
2. 切到 **Scene** 视图  
3. 工具栏打开 **Gizmos**（按钮亮起）  
4. Hierarchy 点选一只 `Enemy_Grunt`（不要选 Brute 也行）  
5. Scene 镜头拉高俯视，能同时看到玩家与该敌人，以及黄/红圈

**画面含义（你要演出来）**

| 状态 | 你怎么做 | Gizmos / 标签 |
|------|----------|----------------|
| Patrol | 人站在 **黄圈外** 等待 2～3 秒 | 敌人游荡；头顶 `AI: Patrol` |
| Chase | WASD **走进黄圈** | 敌人转向追来；`AI: Chase` |
| Attack | 再走进 **红圈** | 停步攻击；`AI: Attack` |
| （加分） | 再跑出黄圈外较远 | 可回到 Patrol（若来得及） |

**颜色说明（旁白可念）**  
- 黄圈：索敌 detectRange  
- 红圈：攻击 attackRange  
- 青圈：巡逻半径  

**字幕**  
`显式有限状态机：Patrol → Chase → Attack`  
`Scene Gizmos：黄=索敌 红=攻击 —— 状态切换看得见`

**旁白**  
「敌人 AI 不是糊在一个 Update 里的 if-else 堆砌，而是显式状态机：巡逻、发现玩家后追击、进入攻击距离后攻击。录 Scene 视图并把索敌、攻击范围画出来，是为了证明状态切换是可调试、可解释的。」

**注意**  
- 状态切换瞬间是精华：走进黄圈那一下务必拍清  
- 若标签看不清，可短暂选中敌人让 Inspector 也入镜（次选）

### 【B】0:48–1:28｜技能编辑器一镜到底（职责2 · 全片最长 · 禁止剪断操作链）

**价值字幕（整段常驻或段首 3 秒大字）**  
`无需修改代码即可配置新技能，降低策划与程序的协作成本`  
`Configure new skills without code changes — lower GD / programmer iteration cost.`

**一镜操作顺序（中间不要切到别的软件）**

1. **Stop** Play（回到编辑模式）  
2. 顶栏点击：`ARPG Tools` → `Skill Editor`（不要双击 .cs 文件）  
3. 若左侧为空：先 `Create Sample Data`，再打开 Skill Editor  
4. 点击工具栏 **Create Skill** → Project 里出现新资产，左侧列表选中它  
5. 右侧把 **Base Damage** 从低拖到高（如 10 → 80），指向底部柱状图变高、柱顶数字变大  
6. 再拖 **Cooldown**、**Range**，指向下方蓝色/绿色进度条变化  
7. （可选）改 Display Name 为 `Demo_FireSlash`，方便口播  
8. 指一下 `VFX Prefab` / `SFX` 对象槽（没有资源也要扫过，说明可挂接）  
9. 不关闭编辑器窗口的情况下：点 Unity **Play**，用 **Q/E/J** 打一下，口头说明「刚改的数值进入运行时」（若新技能未挂到玩家列表，可改为：选中已有 `Skill_Cleave`，把伤害拖高，Play 后用 Q 验证手感变化——**更稳妥的验证路径**）

**更稳妥的「运行时验证」做法（推荐按此录）**

1. Skill Editor 左侧选中 **Skill_Cleave**（已存在的样本）  
2. 记住当前 Base Damage，拖到明显更大（例如 22 → 60）  
3. 保存感（改 SO 会脏标记；无需特殊 Save 面板）  
4. 不关窗口 → Play → 按 **Q** → 敌人掉血明显加快  
5. Stop → 把数值改回或保持均可  

**旁白（配合操作念，不要抢点击）**  
「这是岗位描述里的编辑器和工具开发。完整链路是：新建或选中技能资产，用滑块改伤害和冷却，预览图即时反馈，保存的是 ScriptableObject，运行时执行器直接读同一份数据。全程不用改 C# 逻辑代码。」

**禁止事项**  
- ❌ 中间插入剪辑跳过「拖滑块」  
- ❌ 只放编辑器截图假装操作过  
- ❌ 双击打开 `SkillEditorWindow.cs` 当「打开编辑器」

### 【C】1:28–2:00｜关卡编辑器所见即所得（职责2 · 第二条）

**价值字幕**  
`Scene 里放置的位置 = 运行时 LevelData 生成的位置`  
`What you place is what runtime spawns.`

**一镜操作顺序**

1. Stop Play  
2. `ARPG Tools` → `Level Editor`  
3. Level Data 槽：选 `Assets/Data/Levels/Level_01`（或点 Create Level Data 新建）  
4. 上方 Enemy Types 点选 **Grunt**  
5. 点击 **Place Enemy**（按钮高亮）  
6. 切到 **Scene**，在地面空白处 **左键点 2～3 次** 放置（可见红球 Handle / 列表增加）  
7. 勾选 Has Goal Point → 点 **Place Goal** → Scene 再点一次（绿圈 GOAL）  
8. 点击 **Save Level Asset**  
9. Hierarchy 新建空物体，命名 `LevelLoader`，Add Component → `LevelLoader`  
10. 把刚保存的 LevelData 拖进 Level Data 字段；勾选 Load On Start  
11. （避免叠怪）选中 `DemoBootstrap`，取消勾选 `Build If Scene Empty`，或使用空场景只留相机地面+玩家——**若怕翻车，简化为：**  
    - 演示完放置与 Save 后，Play 已有 DemoBootstrap 战斗即可；口播「LevelLoader 读取同一份 LevelData 生成」并快速展示 LevelLoader 组件已赋值（仍算闭环）  
12. **完整闭环加强版（推荐有时间就做）**：新建场景 `LevelRuntimeTest`，只放地面、光、相机、带 Player 的简易物体或保留 Bootstrap 关敌人，挂 LevelLoader 指到你刚保存的关卡 → Play → 敌人出现在你点击过的位置  

**旁白**  
「关卡编辑器同样是工具链闭环：在 Scene 视图点选放置敌人与目标点，写入 LevelData 资产；运行时 LevelLoader 按资产坐标生成。策划改关卡不需要程序员改场景硬摆。」

**禁止事项**  
- ❌ 放置与 Play 验证之间跳切到毫不相干画面  
- ❌ 不 Save 就说「已经保存」

### 【D】2:00–2:15｜视觉 / 性能 / 图形学（职责3 + 加分项）

**画面组合（可剪，建议慢动作）**

1. Play 中故意挨打 / 打敌人：受击 **闪色 + 描边变粗**（放慢 0.5x）  
2. 击杀：溶解从完整到消失（放慢）  
3. （可选静帧插入 2 秒）Game 视图 Stats：Batches；或 PPT：Profiler GC 左右对比框（数字必须你本机真实截取）

**字幕**  
`描边：第二 Pass、Cull Front、沿法线外扩顶点`  
`溶解：噪声阈值 clip + 边缘色；Cutoff 从 0→1`  
`性能：对象池 Queue 复用特效；PropertyBlock 避免拆材质实例`

**旁白（图形学务必念清楚）**  
「描边 Shader 使用额外 Pass，正面剔除，把顶点沿法线推出一圈再铺纯色，得到轮廓。溶解用噪声和 Cutoff 做阈值裁剪，死亡时提高 Cutoff。特效走对象池，用队列做获取和归还，降低 Instantiate 带来的 GC；受击变色用 MaterialPropertyBlock，避免 renderer.material 拆出新实例抬高 Draw Call。」

**PPT 备选页（可不进视频，可另传附件）**  
- 左：优化前 GC Alloc 曲线图  
- 右：优化后曲线图  
- 标注真实数值与测试步骤（狂按技能 10 秒）

## 三、口播全文（可直接录音棚念 · 约 320 字中文版）

> 大家好，这是我的 Unity 客户端与工具向作品集。  
> 首先是数据驱动的战斗：移动、普攻、技能、受击和溶解死亡。技能的伤害、冷却、范围都在 ScriptableObject 里，执行器只负责读取和执行，不硬编码。  
> 敌人 AI 是显式状态机：巡逻、追击、攻击；Scene 里用 Gizmos 画出索敌和攻击范围，状态切换一目了然。  
> 接下来是重点：自定义编辑器。技能编辑器里可以新建技能、拖滑块改数值、即时预览，不用改代码就能配置，降低策划和程序协作成本。关卡编辑器在 Scene 点击放置敌人，保存为 LevelData，运行时按同样坐标生成，形成所见即所得。  
> 视觉上使用 URP 后处理、法线外扩描边和噪声溶解；性能上特效走对象池，受击用 PropertyBlock 控制合批。  
> 以上是数据层、逻辑层与编辑器层的分工。谢谢。

（英文短版 45 秒可另录，见仓库 `Docs/INTERVIEW_TALKING_POINTS.md`。）
